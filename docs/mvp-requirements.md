# AI Note MVP Requirements

## 1. 제품 개요

AI Note는 스티키노트형 메모 앱의 장점인 빠른 기록 경험을 유지하면서, AI를 통해 메모 정리와 과거 메모 탐색을 개선하는 간단한 메모 앱이다.

- 1차 타깃 사용자: 직장인
- 플랫폼 우선순위: 데스크톱 > 모바일 > 웹
- MVP 핵심 가치: 빠르게 적고, AI로 다듬고, 맥락으로 다시 찾는다

## 2. 해결하려는 문제

기존 경량 메모 앱은 즉시 기록에는 적합하지만, 시간이 지난 뒤 원하는 옛 메모를 다시 찾기 어렵다.

사용자는 다음 문제를 겪는다.

- 해야 할 일, 전화 메모, 업무 메모를 빠르게 적어둘 수는 있다.
- 시간이 지나면 어떤 메모에 무엇을 적었는지 기억하기 어렵다.
- 관련 메모를 다시 찾는 데 시간이 걸린다.
- 방금 쓴 거친 메모를 바로 전달 가능한 문장으로 다듬는 데 추가 수고가 든다.

이 제품은 이 문제를 해결하기 위해, 단순한 메모 UX는 유지하되 AI를 통해 정리와 탐색을 강화한다.

## 3. 대상 사용자

### 3.1 1차 타깃

직장인

### 3.2 사용자 특성

- Notion, Obsidian 같은 도구를 무겁거나 어렵게 느낀다.
- 복잡한 지식관리보다 빠른 기록과 빠른 재탐색이 중요하다.
- 업무 중 메모를 자주 남기지만, 나중에 다시 찾는 과정이 번거롭다.

### 3.3 대표 사용 시나리오

- 전화 중 핵심 내용을 급하게 메모한다.
- 업무 중 해야 할 일을 짧게 적어둔다.
- 나중에 “오늘 할 일”, “아까 전화한 내용”, “클라이언트 관련 메모”처럼 맥락으로 관련 메모를 찾는다.
- 거칠게 적은 내용을 외부에 전달 가능한 문장으로 다듬는다.
- 공손한 문체가 필요한 상황에서 기존 메모를 더 예의 있게 변환한다.

## 4. 핵심 가치 제안

“스티키노트처럼 빠르게 적고, AI로 정리하고, 맥락으로 다시 찾을 수 있는 간단한 메모 앱”

핵심 가치 요소:

- 빠른 기록
- 쉬운 재탐색
- 바로 쓸 수 있는 AI 정리 결과
- 복잡한 도구 대비 낮은 진입장벽

## 5. MVP 범위

### 5.1 이번 출시에서 반드시 포함할 것

- 메모 CRUD
- AI 메모 정리
- 맥락 기반 검색
- 데스크톱 우선 UX

### 5.2 이번 출시에서 제외할 것

- 검색 결과 기반의 AI 답변 생성
- 전체 메모에서 원하는 내용을 추출해 별도 결과물로 제시하는 기능
- 제목 자동 생성
- 협업, 공유, 댓글, 권한 관리
- 첨부파일
- 복잡한 폴더 / 워크스페이스 구조
- 동기화
- 모바일 / 웹 본격 대응
- 추가 AI 기능

## 6. 사용자 시나리오

### 시나리오 1. 빠른 업무 메모 작성

사용자는 업무 중 새 메모를 빠르게 생성하고 내용을 저장할 수 있어야 한다.

### 시나리오 2. 메모 문장 다듬기

사용자는 작성한 메모를 선택해 문장을 더 자연스럽고 읽기 좋게 정리할 수 있어야 한다.

### 시나리오 3. 공손한 문체 변환

사용자는 메모 내용을 공손한 문체로 변환해 메일이나 전달 문구 초안으로 활용할 수 있어야 한다.

### 시나리오 4. 맥락 기반 검색

사용자는 자연어 표현으로 검색해 관련 메모 목록을 빠르게 찾을 수 있어야 한다.

예:

- “오늘 할 일”
- “아까 전화한 내용”
- “클라이언트 관련 메모”

## 7. 기능 요구사항

### FR-1 메모 생성

- 사용자는 새 메모를 생성할 수 있어야 한다.
- 메모는 제목과 본문을 가질 수 있어야 한다.

### FR-2 메모 조회

- 사용자는 메모 목록을 볼 수 있어야 한다.
- 사용자는 특정 메모를 선택해 내용을 볼 수 있어야 한다.

### FR-3 메모 수정

- 사용자는 기존 메모의 제목과 본문을 수정할 수 있어야 한다.

### FR-4 메모 삭제

- 사용자는 메모를 삭제할 수 있어야 한다.

### FR-5 AI 메모 정리

- 사용자는 메모 본문에 대해 AI 정리 기능을 실행할 수 있어야 한다.
- AI 정리 결과는 문장 다듬기와 공손한 문체 변환에 초점을 맞춰야 한다.
- 사용자는 정리 결과를 확인하고 반영할 수 있어야 한다.

### FR-6 맥락 기반 검색

- 사용자는 자연어 또는 맥락성 표현으로 메모를 검색할 수 있어야 한다.
- 검색 결과는 관련 메모 목록이어야 한다.
- 검색 결과는 AI가 합성한 하나의 답변이 아니어야 한다.

### FR-7 검색 결과 정보

- 각 검색 결과는 사용자가 빠르게 판단할 수 있는 최소 정보를 포함해야 한다.
- 포함 후보 정보:
  - 제목
  - 본문 일부
  - 최근 수정 시점

### FR-8 플랫폼 우선 지원

- 데스크톱 경험을 우선 설계해야 한다.
- 모바일은 2순위, 웹은 3순위로 본다.

## 8. 비기능 요구사항

### NFR-1 검색 속도

- 사용자는 원하는 메모를 4초 이내에 찾을 수 있어야 한다.

### NFR-2 AI 정리 품질

- AI 정리 결과는 거의 수정 없이 바로 사용할 수 있어야 한다.

### NFR-3 사용성

- 제품은 스티키노트 앱처럼 단순해야 한다.
- 핵심 작업인 생성, 수정, 검색, AI 정리는 학습 없이 수행 가능해야 한다.

### NFR-4 안정성

- 기본 CRUD는 데이터 손실 없이 일관되게 동작해야 한다.

## 9. 비범위

- 전체 메모를 종합한 AI 답변 생성
- 검색 결과 요약 생성
- 제목 자동 생성
- 협업 / 공유 / 권한 관리
- 동기화
- 일정 관리 / 프로젝트 관리 기능
- 메일 자동 발송
- 복잡한 문서형 편집기

## 10. Acceptance Criteria

- 사용자는 데스크톱 환경에서 메모를 생성, 조회, 수정, 삭제할 수 있다.
- 사용자는 메모 정리 기능을 실행해 문장 다듬기 또는 공손한 문체 변환 결과를 받을 수 있다.
- 사용자는 AI 정리 결과를 거의 수정하지 않고 바로 사용할 수 있다고 느껴야 한다.
- 사용자가 맥락 기반 질의를 입력하면 관련 메모 목록이 반환되어야 한다.
- 검색 결과는 합성 답변이 아니라 메모 리스트여야 한다.
- 사용자는 원하는 메모를 4초 이내에 찾을 수 있어야 한다.

## 11. 열린 이슈 / 가정

### 열린 이슈

- “4초 이내”의 측정 기준
  - 첫 결과 표시 시간인지
  - 사용자가 실제 원하는 메모를 여는 시간인지
- “거의 수정 없이 사용 가능”의 측정 기준
  - 수정 횟수인지
  - 수정 글자 수인지
  - 주관적 만족도인지
- 모바일을 MVP에 포함할지, 후속 단계로 미룰지

### 가정

- 1차 버전은 개인용 메모 사용을 전제로 한다.
- 제목은 사용자가 직접 입력한다.
- 검색은 관련 메모 목록을 보여주는 수준이면 충분하다.
- 추가 AI 기능은 후속 버전으로 미룬다.
